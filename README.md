# Roster

Версия **1.1**. React-приложение для просмотра и фильтрации данных пользователей с использованием [DummyJSON API](https://dummyjson.com/docs/users).

## Важно

**Настоятельно рекомендуется открывать сайт с включённым VPN** — без него из некоторых регионов запросы к API часто зависают. В приложении используется бесплатный фейковый API **DummyJSON**. При ошибке соединения ранее загруженные данные остаются в кэше и отображаются с уведомлением.

## О проекте

React-приложение для работы со списком пользователей:

- **Таблица пользователей** — фамилия, имя, отчество, возраст, пол, телефон, email, страна, город; ширина таблицы до 1400px, изменение ширины колонок (мин. 50px).
- **Сортировка** — по ФИО (фамилия/имя/отчество), возрасту, полу, телефону; три состояния: по возрастанию ↑, по убыванию ↓, без сортировки ⇅. Пустые отчества при сортировке по отчеству отображаются в конце.
- **Фильтрация** — по ФИО (поиск по имени, фамилии, отчеству; несколько слов — все учитываются), возраст от/до, пол, телефон (пробелы и дефисы не учитываются). Все заполненные поля работают вместе (логика И).
- **Постраничная навигация** — 10 пользователей на страницу, кэш страниц и префетч следующей.
- **Модальное окно** — по клику на строку: ФИО, возраст, адрес, рост, вес, телефон, email, аватар.
- **Обработка ошибок** — таймаут запросов, баннер с повтором, подсказка при долгой загрузке (VPN). Данные из кэша не сбрасываются при ошибке.
- **Тема** — переключение светлая/тёмная (localStorage + system preference).

Стек: React 19, TypeScript, Vite, Fetch API, SCSS.

## Функциональность по ТЗ

### Компоненты

- **Filters** — поля фильтрации (ФИО, возраст от/до, пол, телефон) и подсказка по логике.
- **UsersTable** — таблица с сортируемыми заголовками, ресайзом колонок, пустым состоянием «Нет данных».
- **UserModal** — модальное окно с подробной информацией о пользователе (ФИО, возраст, адрес, рост, вес, телефон, email, аватар).
- **Pagination** — информация «Показано X–Y из Z» и кнопки навигации по страницам.
- **ErrorBoundary** — перехват ошибок рендера, fallback-экран.

### API и данные

- **api/users.ts** — `fetchUsers(params)` (пагинация, сортировка, фильтры), `fetchUserById(id)`. При фильтрах или сортировке по отчеству — загрузка с последующей фильтрацией/сортировкой на клиенте.
- **api/fetchWithTimeout.ts** — обёртка над Fetch API с таймаутом и единым сообщением об ошибке.
- **hooks/useUsers** — загрузка списка, кэш по ключу (страница + фильтры + сортировка), префетч следующей страницы, отмена при размонтировании.

### Константы и типы

- **constants/config.ts** — URL API, лимиты, задержки (debounce, подсказка при долгой загрузке), границы возраста.
- **constants/messages.ts** — все пользовательские сообщения (ошибки, подсказки, кнопки, пагинация).
- **constants/layout.ts** — размер страницы, мин/макс ширина колонок, ширина приложения, дефолтные ширины колонок.
- **constants/table.ts** — конфиг колонок таблицы, сортируемые поля, плейсхолдер пустой ячейки.
- **types/user.ts** — тип пользователя и ответа API.
- **types/sort.ts** — типы и константы сортировки (колонка, направление).

### Дополнительно

- Стили: SCSS, переменные в `_variables.scss`, темы через CSS-переменные, медиа-запросы для адаптива.
- Деплой на GitHub Pages через GitHub Actions (workflow в `.github/workflows/deploy.yml`). В настройках репозитория: **Settings → Pages → Source: GitHub Actions**.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/gaminv/user-table.git
cd user-table
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Локальный запуск

```bash
npm run dev
```

Открыть в браузере: [http://localhost:5173](http://localhost:5173)
